# Module 01: Foundations — Completion Checklist

Use this checklist to self-assess before moving to Module 02.

---

## Setup Verification

- [ ] GitHub Copilot Pro+ is active — the Copilot icon is visible in the VS Code status bar
- [ ] Copilot chat panel opens with `Ctrl+Alt+I`
- [ ] Inline completions appear as grey ghost text when typing in a code file
- [ ] `.vscode/settings.json` in this repo is loaded (open a file and confirm completions appear)

---

## Mode Knowledge

- [ ] I can switch between Ask, Edit, Plan, and Agent modes in the chat panel
- [ ] I can trigger inline chat with `Ctrl+I` while the cursor is in the editor
- [ ] I can explain — in one sentence — when I would use each of the six modes
- [ ] I know the keyboard shortcut to cycle through alternative inline completions

---

## Exercise Completion

- [ ] Exercise 1 completed — accepted and dismissed ghost text completions
- [ ] Exercise 2 completed — used Ask mode to explain a function and identify a security issue
- [ ] Exercise 3 completed — used Edit mode to rename a variable and reviewed the diff
- [ ] Exercise 4 completed — used Plan mode before writing code
- [ ] Exercise 5 completed — used inline chat to fix a syntax error and a logic issue

---

## Critical Evaluation

- [ ] I can apply the four critical review questions to any AI-generated code:
  1. Does it compile / run?
  2. Does it do what I asked?
  3. Does it break anything?
  4. Do I understand it fully?
- [ ] I know at least two common AI failure modes (e.g., hallucinated API, missing error handling)
- [ ] I would not commit AI-generated code without reading it line by line

---

## Token Awareness

- [ ] I know which actions consume included requests vs. premium requests
- [ ] I would choose inline completion or standard chat before escalating to agent mode for simple tasks

---

## Ready for Module 02?

If all boxes above are checked, proceed to:

If any are unchecked, revisit the corresponding exercise or the [theory.md](./theory.md) reference section.
