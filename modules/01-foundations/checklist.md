# Module 01: Foundations — Completion Checklist

Use this checklist to self-assess before moving to Module 02. Every item must be checked.

---

## Setup Verification

- [ ] GitHub Copilot Pro+ is active — the Copilot icon is visible in the VS Code status bar without a warning
- [ ] Copilot chat panel opens with `Ctrl+Alt+I`
- [ ] Inline completions appear as grey ghost text when typing in a code file
- [ ] I can confirm I am on the Pro+ plan (chat response mentions Pro+ or an agent-capable plan)

---

## Mode Knowledge

- [ ] I can switch between Ask, Plan, and Agent modes in the chat panel
- [ ] I can trigger inline chat with `Ctrl+I` while the cursor is in the editor
- [ ] I can explain in one sentence when I would use each of the five modes
- [ ] I know the keyboard shortcut to cycle through alternative inline completions (`Alt+]` / `Alt+[`)
- [ ] I understand why beginners should default to Ask and Inline chat for lightweight tasks

---

## Exercise Completion

- [ ] Exercise 1 completed — accepted and dismissed ghost text, cycled alternatives
- [ ] Exercise 2 completed — used Ask mode to explain a function and identify the MD5 security issue
- [ ] Exercise 3 completed — used Agent mode to rename a variable and reviewed the diff before accepting
- [ ] Exercise 4 completed — used Plan mode before writing code and identified at least one edge case
- [ ] Exercise 5 completed — used inline chat to fix a syntax error and add a division-by-zero guard
- [ ] Exercise 6 completed — classified 5 tasks as included or premium requests with written reasoning

---

## Critical Evaluation

- [ ] I can apply Read → Run → Reason → Risk to any AI suggestion
- [ ] I name at least three of the five AI failure modes without looking them up
- [ ] I understand what OWASP A01 (broken access) and A03 (injection) mean in the context of AI output
- [ ] I apply the explain rule: I would not commit AI code I cannot explain in my own words

---

## Token and Cost Awareness

- [ ] I can distinguish included requests from premium requests without looking up the documentation
- [ ] I know the cost-aware decision rule: use the cheapest mode that handles the task; escalate only if needed
- [ ] I know which three factors increase per-turn cost in a chat session

---

## Productivity Habits

- [ ] I choose my Copilot mode before typing a prompt
- [ ] I write a one-sentence acceptance criterion before sending a prompt for non-trivial tasks
- [ ] I close irrelevant files before starting a chat session
