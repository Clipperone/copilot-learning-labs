# Capstone: End-to-End Copilot Workflow Integration

Apply every skill from the course to a single project context. Configure the full Copilot integration layer, design an agent workflow, validate an AI-generated diff, and produce a 90-day adoption plan.

**Level:** Expert · **Estimated time:** ~2 hours

**Prerequisites:** All 11 modules and Labs 01–09 complete.

---

## Required Deliverables

| # | Deliverable | Module |
|---|-------------|--------|
| 1 | `.github/copilot-instructions.md` — ≥5 rules in declarative format | M05 |
| 2 | `CONVENTIONS.md` — governance policy with `AI-assisted:` and `Reviewed-by:` trailers | M09 |
| 3 | `agents/` — ≥3 agent definitions with purpose, constraints, tool permissions, and handoffs | M06 |
| 4 | `prompts/` — ≥3 prompt entries in the course template format with filled examples | M04 |
| 5 | `agents/workflow-[name].md` — one multi-agent workflow with all 5 required sections | M07 |
| 6 | `capstone/validation-report.md` — gate-by-gate pre-merge validation | M08/M09 |
| 7 | `capstone/roadmap.md` — 90-day adoption plan with milestones and KPIs | M10 |
| 8 | `capstone/platform-artifact.md` — one platform artifact (Option A: coding-agent PR + review notes; Option B: `copilot-setup-steps.yml` + rewritten PR summary) | M11 |
