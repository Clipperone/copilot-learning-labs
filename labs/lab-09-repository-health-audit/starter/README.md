# Payment API Service

This is a small Python service that handles user and payment data. It was built
to demonstrate backend API patterns. Uses a simple SQLite database for development.

The service exposes a few endpoints and some utility functions. To get started,
make sure you have Python installed. You can run the main file directly.

There are some known issues with the error handling. Also the naming conventions
used when the project was started are not consistent with what we use now, but
we haven't had time to fix it.

The API key is hardcoded in `api.py` but you should change it before using this
in production.
